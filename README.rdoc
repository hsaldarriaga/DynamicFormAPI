== README

Conceptos

Categorias -> Groups

Procedures -> Procesos

Steps -> Pasos

Contiene un campo llamado Step(step_id) que es el orden del paso según la secuencia
Procedure (procedure_id) es el id del proceso al cual está ligado

Content: Contiene los campos que contiene el paso, las decisiones que se van a tomar y hacía donde van las decisiones

La estructura de las categorías es:

== GROUPS: 
<p>Id: Identificador único usado por Ruby on Rails.</p>
<p>Group_id: Identificador del grupo.</p>
<p>Name: Nombre del grupo</p>
<p>URL: Enlace hacia el JSON de este grupo.</p>

== PROCEDURES:
<p>Id: Identificador único asignado por Rails.</p>
<p>Procedure_id: Identificador del procedimiento.</p>
<p>Group_id: Identificador del grupo al cual pertenece el procedimiento.</p>
<p>Name: Nombre del procedimiento.</p>
<p>Description: Descripción del procedimiento.</p>
<p>URL: Enlace hacia este JSON de este procedimiento.</p>

== STEPS:
<p>ID: Identificador único asignado por Rails.</p>
<p>Step_id: Identificador del paso.</p>
<p>Procedure_id: Identificador del procedimiento del cual precede.</p>
<p>Content: Es una cadena que contiene el siguiente Json</p>
<p>Fields: Campos que tiene el step</p>
<p>Decisions: Conjunto de decisiones que conducen a otro step.</p>
<p>FIELDS: Es un vector de campos</p>
<p>Id :  Identificador del campo</p>
<p>Caption: Aviso que tiene el campo</p>
<p>Field_type: Tipo de Campo: 0, Questions. 1, boolean. 2, Numeric, 3, Label</p>
<p>Posible_values: Posibles valores del campo</p>

== DECISIONS: 
<p>Branch: contiene una rama de decisiones según los campos</p>
<p>Field_id = identificador del campo</p>
<p>Comparision_type: Tipo de comparación, puede ser >, = y <</p>
<p>Value = Valor que va a ser comparado</p>
<p>go_to_step: step_id al cual se dirige si se sigue la rama de decisión.</p>
